# Endpoints y servicios

Esta pagina sirve como indice para documentar APIs por dominio.

## Convencion propuesta

Para cada endpoint documentar:

- Metodo y ruta
- Autenticacion/autorizacion requerida
- Request body o query params
- Respuesta exitosa
- Errores esperados

## Plantilla sugerida

~~~mdx
### POST /api/ejemplo

**Descripcion:** Crea un recurso.

**Auth:** Requerida (rol admin).

**Request:**
```json
{ "name": "valor" }
```

**Response 200:**
```json
{ "id": "123", "name": "valor" }
```
~~~
