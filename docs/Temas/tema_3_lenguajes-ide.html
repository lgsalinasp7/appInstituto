MÓDULO 01

Historia de los Lenguajes

Desde el lenguaje ensamblador hasta la IA generativa — un viaje de 70 años

1940s – 1950s
🔌 Lenguaje Máquina & Ensamblador
Los primeros computadores se programaban con 0s y 1s directamente. El ensamblador fue el primer paso hacia instrucciones más humanas: MOV AX, 01h
Lenguaje Máquina
1954
📐 FORTRAN — El primero de alto nivel
IBM crea el primer lenguaje de alto nivel para cálculos científicos y matemáticos. Por primera vez los programadores podían escribir algo parecido al álgebra.
IBM · 1954
1959
💼 COBOL — Lenguaje de negocios
Grace Hopper lidera el desarrollo de COBOL para aplicaciones comerciales. Todavía hoy miles de millones de transacciones bancarias corren en COBOL.
Grace Hopper
1972
⚙️ C — La base de todo
Dennis Ritchie en Bell Labs crea C. Es el padre de casi todos los lenguajes modernos: C++, Java, Python, JavaScript y más. Linux y Unix están escritos en C.
Bell Labs · Dennis Ritchie
1983
🏗 C++ — Orientado a Objetos
Bjarne Stroustrup añade orientación a objetos a C. Los videojuegos, motores gráficos y sistemas operativos se construyen en C++.
POO · Stroustrup
1991 – 1995
🚀 Python & JavaScript nacen
Guido van Rossum crea Python (1991) por su legibilidad. Brendan Eich crea JavaScript en 10 días (1995) para hacer las páginas web interactivas.
Era Web · 1990s
1995
☕ Java — Write Once, Run Anywhere
Sun Microsystems lanza Java con la promesa de ejecutarse en cualquier plataforma. Domina el desarrollo empresarial y Android por décadas.
Sun Microsystems
2000s
🌐 La era de la Web
PHP, Ruby, ASP.NET proliferan para el desarrollo web. Rails (2004) revoluciona la productividad. AJAX hace las páginas dinámicas sin recargar.
Web 2.0
2012
🔷 TypeScript — JS con tipos
Microsoft lanza TypeScript para añadir tipado estático a JavaScript. Se convierte en el estándar para proyectos grandes y escalables.
Microsoft · Anders Hejlsberg
2015 – Hoy
🤖 Era de la Inteligencia Artificial
Python domina la IA con TensorFlow, PyTorch y scikit-learn. Rust, Go y Kotlin toman fuerza. Los LLMs como GPT y Claude cambian cómo programamos.
IA · LLMs · 2024+
MÓDULO 02

Tipos de Lenguajes

Clasificaciones clave que todo desarrollador debe entender

⚙️
Bajo Nivel
Muy cercanos al hardware. Ofrecen control total pero son difíciles de escribir. Un error puede corromper la memoria del sistema.
Ensamblador Lenguaje Máquina
; Hola mundo en Ensamblador x86
section .data
  msg db 'Hola, Mundo!', 0Ah
section .text
  mov eax, 4
  int 0x80
🔧
Nivel Medio
Balance entre control del hardware y legibilidad. C y C++ permiten manejar memoria directamente pero con una sintaxis más humana.
C C++ Rust
// C — Hola Mundo
#include <stdio.h>
int main() {
  printf("Hola, Mundo!\n");
  return 0;
}
✨
Alto Nivel
Muy cercanos al lenguaje humano. El programador no gestiona memoria manualmente. Son más lentos que C pero mucho más productivos.
Python JavaScript Java Ruby PHP
# Python — tan simple como inglés
print("Hola, Mundo!")

# Una línea es todo lo que necesitas
📋
Imperativo / Procedural
Le dices al computador CÓMO hacer algo, paso a paso. El flujo de control es explícito: si esto, entonces aquello.
CPascalBASIC
🧩
Orientado a Objetos (POO)
Organiza el código en objetos que tienen propiedades y comportamientos. Simula el mundo real: un Carro tiene motor, color y puede acelerar.
JavaC++PythonRuby
class Carro:
  def __init__(self, color):
    self.color = color
  def acelerar(self):
    print("¡Vroom!")
λ
Funcional
Las funciones son ciudadanos de primera clase. No hay estado mutable. Ideal para procesar datos y evitar efectos secundarios.
HaskellErlangClojureElixir
🔍
Declarativo
Le dices QUÉ quieres, no cómo obtenerlo. SQL es el ejemplo perfecto: describes el resultado y la base de datos decide cómo obtenerlo.
SQLHTMLCSSProlog
-- SQL: qué quiero, no cómo
SELECT nombre, edad
FROM usuarios
WHERE edad > 18
ORDER BY nombre
🔒
Tipado Estático
Los tipos se definen y verifican en tiempo de compilación. Más errores detectados antes de ejecutar. Más verboso pero más seguro.
JavaC#TypeScriptRustGo
// TypeScript — tipos explícitos
let edad: number = 25;
let nombre: string = "Ana";
// Error en tiempo de compilación:
edad = "hola"; // ❌ Error!
🔓
Tipado Dinámico
Los tipos se verifican en tiempo de ejecución. Más flexible y rápido para escribir, pero los errores aparecen cuando el programa corre.
PythonJavaScriptRubyPHP
# Python — flexible
x = 10      # número
x = "hola"  # ahora string
x = [1,2,3] # ahora lista ✓
🧠
Tipado Fuerte vs Débil
Fuerte: no hace conversiones automáticas de tipo. Débil: convierte tipos automáticamente (puede causar bugs inesperados).
Python (fuerte)JavaScript (débil)
// JS débil — suma rara:
1 + "2" = "12" // concatena!

# Python fuerte:
1 + "2" # ❌ TypeError
⚡
Compilados
El código fuente se traduce COMPLETO a lenguaje máquina antes de ejecutarse. Muy rápidos en ejecución. El binario resultante es independiente.
CC++RustGo
código.c → compilador → programa.exe
🔄
Interpretados
El código se ejecuta línea por línea en tiempo real por un intérprete. Más lentos pero portables y fáciles de depurar. No necesitan compilación.
PythonRubyPHPBash
código.py → intérprete → resultado
🌉
Compilados a Bytecode (JVM)
Se compilan a un código intermedio (bytecode) que corre en una máquina virtual. Combina portabilidad y velocidad relativa.
JavaKotlinScalaC#
App.java → .class bytecode → JVM → ejecución
🏎
JIT (Just-In-Time)
Compilan en tiempo de ejecución las partes más usadas del código. JavaScript moderno usa V8 con JIT — por eso es tan rápido hoy.
JavaScript (V8)Python (PyPy)Java HotSpot
MÓDULO 03

La Historia de JavaScript

De 10 días de desarrollo a dominar el mundo del desarrollo moderno

1995
⚡ Nacimiento en 10 días
Brendan Eich crea JavaScript en solo 10 días para Netscape. Su objetivo: hacer las páginas web interactivas. Originalmente se llamó "Mocha", luego "LiveScript".
JS
ES
1997
📋 ECMAScript — El estándar
ECMA International estandariza JavaScript como ECMAScript. Esto unifica las implementaciones entre Netscape e Internet Explorer. ES1 es el punto de partida.
2005
🔄 AJAX — La revolución
AJAX (Asynchronous JavaScript and XML) permite actualizar partes de la página sin recargarla. Gmail y Google Maps demuestran el poder. La web 2.0 nace.
// Fetch sin recargar la página
fetch('/api/datos')
  .then(res => res.json())
  .then(data => render(data));
AX
V8
2008
🚀 Motor V8 de Google
Google Chrome lanza el motor V8 con compilación JIT. JavaScript pasa de "lento" a increíblemente rápido. Esto cambia todo lo que vendrá después.
2009
🟢 Node.js — JS en el servidor
Ryan Dahl toma V8 y lo pone en el servidor. Por primera vez un mismo lenguaje corre en frontend Y backend. El desarrollo full-stack con JS se vuelve posible.
const http = require('http');
http.createServer((req, res) => {
  res.end('¡Hola desde Node!');
}).listen(3000);
N
Re
2013
⚛️ React — UI declarativa
Facebook lanza React. Los componentes reutilizables y el Virtual DOM revolucionan cómo se construyen interfaces. El modelo mental cambia para siempre.
function Saludo({ nombre }) {
  return <h1>Hola, {nombre}!</h1>;
}
2015
🎯 ES6 / ES2015 — El gran salto
La mayor actualización de JS: arrow functions, clases, módulos, promesas, destructuring, template literals, let/const. El lenguaje se moderniza completamente.
// ES6: mucho más limpio
const sumar = (a, b) => a + b;
const [x, y] = [1, 2];
const msg = `Hola ${nombre}`;
6+
TS
2016 – 2019
🔷 TypeScript domina
Microsoft's TypeScript se adopta masivamente. Angular lo adopta como lenguaje oficial. Vue y React siguen. Los proyectos grandes ya no se conciben sin tipos.
2016
▲ Next.js — El meta-framework
Vercel lanza Next.js sobre React. SSR (Server Side Rendering), SSG, routing automático, API Routes. Se convierte en el estándar para aplicaciones React de producción.
// pages/index.tsx — Next.js
export default function Home() {
  return <h1>Mi App Next.js</h1>
}
// SSR automático ✓ SEO ✓
NX
AI
2023 – Hoy
🤖 JS + IA — El futuro
Vercel AI SDK, LangChain.js, OpenAI SDK. JavaScript lidera la integración de IA en el frontend. Los Server Components de Next.js 13+ cambian la arquitectura web.
// Next.js + IA (Vercel AI SDK)
import { useChat } from 'ai/react';
export default function Chat() {
  const { messages, input } = useChat();
}

🏆 Ecosistema JavaScript hoy (2024)

JavaScript es el lenguaje más popular del mundo por 12 años consecutivos (Stack Overflow Survey). Node.js, React, Vue, Angular, Next.js, Nuxt, SvelteKit, Bun, Deno — el ecosistema es enorme. TypeScript es ahora el estándar para proyectos serios. Y con la explosión de la IA, JavaScript/TypeScript es el lenguaje #1 para integrar LLMs en aplicaciones web.

MÓDULO 05

Historia de los IDEs

De editar en Notepad a que la IA escriba código por ti

¿Qué es un IDE?

Un Integrated Development Environment (IDE) es un programa que combina todo lo que necesitas para programar: editor de código, depurador, terminal, control de versiones y más — todo en una sola ventana. Es la diferencia entre escribir código en un bloc de notas versus tener un copiloto inteligente.

1970s – 1980s
📟 Los editores de texto
Vi (1976) y Emacs (1976) nacen en Unix. No eran IDEs pero sí poderosos editores. Sus atajos de teclado siguen siendo referencia hoy. "Salir de Vim" sigue siendo un meme.
Vi · Emacs · 1976
1991
🖼 Turbo Pascal — El primer IDE moderno
Borland crea Turbo Pascal con una interfaz visual, depurador integrado y compilación instantánea. Es considerado el primer IDE verdadero. Revoluciona la productividad.
Borland · GUI
1997 – 2001
☕ Eclipse & NetBeans
IBM lanza Eclipse (2001), Sun NetBeans (1997). IDEs gratuitos y open source para Java que establecen el estándar: proyectos, depurador, refactoring, plugins.
Java IDEs
2000 – 2008
🏢 Visual Studio & Xcode
Microsoft Visual Studio domina el desarrollo Windows/C#. Apple lanza Xcode para Mac/iOS. Los IDEs se vuelven masivos — miles de MB, pero increíblemente potentes.
Microsoft · Apple
2008 – 2013
🧠 JetBrains — IDEs inteligentes
IntelliJ IDEA, PyCharm, WebStorm. JetBrains reinventa los IDEs con autocompletado profundo, análisis estático y refactoring avanzado. "¿Cómo lo sabe?" era la reacción común.
JetBrains
2015
💙 VS Code — La revolución ligera
Microsoft lanza Visual Studio Code. Open source, gratuito, rápido. En 3 años se convierte en el editor #1 del mundo. El marketplace de extensiones cambia todo.
Microsoft · Open Source
2020 – 2023
☁️ IDEs en la nube
GitHub Codespaces, GitPod, Replit, StackBlitz. Programar desde el navegador sin instalar nada. La pandemia acelera su adopción. El IDE viaja en la URL.
Cloud IDEs
2021 – Hoy
🤖 IDEs con IA — El presente
GitHub Copilot (2021) integra IA en VSCode. Cursor, Windsurf, y Claude Code llevan la IA al centro del flujo de trabajo. Ya no autocompletamos — la IA escribe funciones completas.

El editor open source que se convirtió en plataforma para la siguiente generación de IDEs con IA

¿Qué es un "Fork"?

VS Code es open source (MIT License). Cualquiera puede tomar su código, modificarlo y lanzar su propio editor. Esto creó un ecosistema de "forks" — editores que heredan la base de VS Code pero añaden características propias, especialmente inteligencia artificial.

MÓDULO 07

IDEs de IA Pura

La nueva generación de herramientas donde la IA NO es un plugin — es el núcleo

🚀 El paradigma cambió

Los IDEs tradicionales son editores de texto con IA añadida encima. La nueva generación invierte esto: son sistemas de IA que también tienen editor de texto. Algunos ni siquiera tienen interfaz visual — operan desde la terminal como agentes autónomos.

CC
Claude Code
El agente de terminal de Anthropic

Claude Code es un agente de IA que corre directamente en tu terminal. No tiene interfaz visual — es una CLI (Command Line Interface). Le dices qué quieres y él navega archivos, escribe código, ejecuta tests y hace commits.

  • Sin interfaz visual — 100% terminal
  • Entiende todo el contexto del proyecto
  • Hace commits, crea ramas, abre PRs
  • Ejecuta tests y los corrige automáticamente
  • Navega y edita múltiples archivos
  • Creado por Anthropic (creadores de Claude)
CLI · Agentic · Anthropic

  • Integrado en VS Code y JetBrains
  • Modo agente: múltiples pasos autónomos

🎯 La conclusión para estudiantes

No hay una herramienta "correcta" — depende de tu flujo de trabajo. Comienza con VS Code porque es la base de todo el ecosistema. Cuando seas cómodo con él, prueba Cursor o Windsurf para ver el poder de la IA integrada. Si quieres entender los agentes de terminal, instala Aider o prueba Claude Code. El futuro del desarrollo es IA + humano — y estas herramientas son el presente de ese futuro.
